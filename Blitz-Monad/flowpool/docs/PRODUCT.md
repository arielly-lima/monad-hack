# Product

FlowPool treats a Pix payment as the start of continuous programmable liquidity. Funds enter a treasury layer, split into available liquidity, safety reserve, and yield allocation, then keep powering API requests, AI jobs, and agent activity.

## User problem

- Pix top-ups are static once paid.
- API and AI usage need continuous funding, not manual balance management.
- Agentic systems need auditable budgets, payment rails, and recharge logic.

## Key flows

- Pix inflow to pool activation
- Pay-per-use debits from available liquidity
- Agent budget enforcement and spend tracking
- Threshold-based recharge orchestration
- Simulated background yield on allocated capital
- Monad Testnet write path for real transaction proof
